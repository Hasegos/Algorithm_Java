## 문제

재원이는 한 도시의 시장이 되었다. 이 도시에는 도시를 동쪽과 서쪽으로 나누는 큰 일직선 모양의 강이 흐르고 있다.<br>
하지만 재원이는 다리가 없어서 시민들이 강을 건너는데 큰 불편을 겪고 있음을 알고 다리를 짓기로 결심하였다.<br>
강 주변에서 다리를 짓기에 적합한 곳을 사이트라고 한다. 재원이는 강 주변을 면밀히 조사해 본 결과 강의 서쪽에는 N개의 사이트가 있고 동쪽에는 M개의 사이트가 있다는 것을 알았다. (N ≤ M)

재원이는 서쪽의 사이트와 동쪽의 사이트를 다리로 연결하려고 한다. (이때 한 사이트에는 최대 한 개의 다리만 연결될 수 있다.)<br>
재원이는 다리를 최대한 많이 지으려고 하기 때문에 서쪽의 사이트 개수만큼 (N개) 다리를 지으려고 한다.<br>
다리끼리는 서로 겹쳐질 수 없다고 할 때 다리를 지을 수 있는 경우의 수를 구하는 프로그램을 작성하라.

## 입력

입력의 첫 줄에는 테스트 케이스의 개수 T가 주어진다. 그 다음 줄부터 각각의 테스트케이스에 대해 강의 서쪽과 동쪽에 있는 사이트의 개수 정수 N, M (0 < N ≤ M < 30)이 주어진다.

---

## 출력

각 테스트 케이스에 대해 주어진 조건하에 다리를 지을 수 있는 경우의 수를 출력한다.

## 해결방안
**1.서쪽 사이트 n개와 동쪽 사이트 m개가 있을 때, 다리를 겹치지 않게 놓는 경우의 수는 m개의 위치 중 n개를 선택하는 조합 문제이며 `mCn`으로 계산된다.**<br>
**2.조합 공식 `mCn = m! / (n! (m−n)!)`을 그대로 팩토리얼로 계산하면 데이터가 커질 때 오버플로와 연산량 문제가 발생한다.**<br>
**3.이를 해결하기 위해 (n+1)부터 m까지의 곱을 누적하면서 동시에 1부터 n까지 나눠주는 방식으로 조합을 계산하여 연산량과 overflow를 줄인다.**<br>
**4.각 테스트케이스마다 계산된 조합 값을 출력해 다리를 놓을 수 있는 경우의 수를 구한다.**
